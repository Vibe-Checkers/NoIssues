This page provides technical documentation on how to install ccache and use it effectively in various development environments. It covers installation methods, basic usage patterns, configuration options, and compiler integration techniques. For information about ccache's internal architecture, see System Architecture.
1. Prerequisites
Before installing ccache, ensure your system meets the following requirements:
A C++17 compiler for building from source
CMake 3.15 or newer (if building from source)
Sufficient disk space for the cache (default max size is 5 GB)
Various software library dependencies are also required if building from source:

4. Configuration
ccache can be configured through configuration files, environment variables, and command-line options.
4.1 Configuration Files
The default user configuration file is located at:
Linux/Unix: ~/.config/ccache/ccache.conf or ~/.ccache/ccache.conf
macOS: ~/Library/Preferences/ccache/ccache.conf
Windows: %LOCALAPPDATA%\ccache\ccache.conf
The system-wide configuration file is typically /etc/ccache.conf.

5. Caching Modes
ccache has three main operation modes that affect its performance and reliability:
5.1 Direct Mode (Default)
Hashes source files and header files directly without preprocessing. This is the fastest mode but may miss some dependencies.
5.2 Preprocessor Mode
Runs the preprocessor and hashes its output. This is the most reliable mode but slower than the others.
5.3 Depend Mode
Uses compiler-generated dependency information to identify included files. This mode balances speed and reliability.
